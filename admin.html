The free tier handles approximately 50,000 asset records with photos. Photos are stored as URLs (not embedded base64), which is why you can now register unlimited assets — the database stores a link to the photo, not the photo itself.

Step 1 — Create Supabase Account

Go to supabase.com → Sign Up → Create New Project → choose a region close to your users (e.g. West Africa → use EU West). Note your Project URL and anon/public API key from Settings → API.

Step 2 — Run This SQL in Supabase SQL Editor

Open your Supabase project → SQL Editor → New Query → paste and run:

-- AssetTrack Pro — Complete Database Schema -- Run this entire block in Supabase SQL Editor -- 1. ASSETS TABLE create table if not exists assets ( id text primary key, barcode text unique not null, name text not null, category text, sub_category text, location text, department text, status text default 'Active', condition text default 'Good', value numeric default 0, custodian text, serial text, purchase_date date, notes text, photos text[], -- array of photo URLs from Supabase Storage registered_by text, registered_from_audit boolean default false, last_audit_date timestamptz, last_audit_by text, last_audit_changes text, created_at timestamptz default now(), updated_at timestamptz default now() ); -- 2. USERS TABLE create table if not exists users ( id uuid primary key default gen_random_uuid(), username text unique not null, password_hash text not null, -- use Supabase Auth in production name text, role text default 'field', -- 'admin' | 'field' | 'auditor' | 'viewer' active boolean default true, created_at timestamptz default now() ); -- 3. CONFIG TABLE (taxonomy, locations, departments, org settings) create table if not exists config ( key text primary key, value text not null, updated_at timestamptz default now() ); -- 4. AUDIT LOG TABLE create table if not exists audit_log ( id uuid primary key default gen_random_uuid(), asset_id text references assets(id), barcode text, action text, -- 'verified' | 'flagged' | 'unknown' | 'updated' changes text, audited_by text, audited_at timestamptz default now(), notes text ); -- 5. ROW LEVEL SECURITY (RLS) -- Field users can INSERT assets and UPDATE (audit) but cannot DELETE alter table assets enable row level security; create policy "Field users can insert" on assets for insert with check (true); create policy "Field users can read" on assets for select using (true); create policy "Admins can update" on assets for update using (true); create policy "Admins can delete" on assets for delete using (true); -- 6. STORAGE BUCKET for photos insert into storage.buckets (id, name, public) values ('asset-photos', 'asset-photos', true) on conflict do nothing; create policy "Anyone can upload photos" on storage.objects for insert with check (bucket_id = 'asset-photos'); create policy "Anyone can view photos" on storage.objects for select using (bucket_id = 'asset-photos'); -- 7. DEFAULT ADMIN USER (change password after setup!) insert into users (username, password_hash, name, role) values ('admin', 'admin123', 'System Administrator', 'admin') on conflict do nothing; -- 8. SEED DEFAULT FIELD USER insert into users (username, password_hash, name, role) values ('field.officer', 'field123', 'Field Officer', 'field') on conflict do nothing;
Step 3 — Add Your Keys to Both HTML Files

Open both field.html and admin.html in a text editor and replace:

// Find these two lines near the top of the <script> section: const SUPABASE_URL = 'YOUR_SUPABASE_URL'; const SUPABASE_ANON_KEY = 'YOUR_SUPABASE_ANON_KEY'; // Replace with your actual values, e.g.: const SUPABASE_URL = 'https://abcdefghijkl.supabase.co'; const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';

Then re-upload both files to GitHub. Your data now saves to the cloud permanently. All field officers' registrations will appear in the admin dashboard instantly.
